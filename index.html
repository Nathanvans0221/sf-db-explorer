🔒 Permission Tiers

Three levels of access with safety guardrails at every level

Explorer (Default)

Read-Only Browsing

Browse, search, count, and report on anything. No risk of changing data.

  • SELECT queries across all databases and tables
  • Automatic TOP 50 limits to prevent overwhelming results
  • Clean markdown tables, formatted dates, and commas on numbers
Editor (Type "elevate")

Supervised Data Changes

UPDATE and INSERT on approved tables. Every change goes through a preview → confirm flow.

  • Must type "I understand" to enable
  • Always previews affected rows before executing
  • Major changes (5+ rows, sensitive tables) require Nathan's Teams approval
  • Type "read-only" to drop back anytime
Admin (Nathan Only)

Full Write Access

Adds DELETE capability. Same preview → confirm flow, but no approval routing needed.

  • Single-row DELETE with WHERE clause only
  • Extra confirmation required for 5+ row deletions
  • DROP, ALTER, TRUNCATE are still blocked for everyone

❓ FAQ

What databases can I connect to? ▶
Any Silver Fern customer database server. The hostname follows the pattern {customer-code}.silverfern.app. Each server typically has Produce, Restock, Analyze, and MobileApp databases. Ask Nathan or your team lead for the credentials.
Can I accidentally break something? ▶
By default you're in read-only Explorer mode — you literally cannot change any data. If you need to edit, you must explicitly escalate, and every change shows you exactly what will happen before it executes. Major changes require Nathan's approval via Teams.
Do I need to know SQL? ▶
Nope. Just ask questions in plain English like "show me all customers" or "what's scheduled for week 20?" Claude handles the SQL translation. You can also use quick commands like tables, sample [tablename], or count [tablename].
Does this store my database password? ▶
No. Credentials exist only in the current Claude Code session. When you close the terminal, they're gone. Nothing is written to disk.
What if pymssql won't install? ▶
On most systems pip3 install --break-system-packages pymssql works. If you're on macOS, you may need brew install freetds first. On Windows/WSL it usually just works. Ask in the Product chat if you get stuck.
